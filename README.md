# Lecture Notes Viewer

Upload PDF slides + a transcript, get back beautifully rendered Markdown lecture notes (and PDFs) generated by Claude. Runs locally on your Mac.

## Install on a fresh Mac

The only prerequisite is **Xcode Command Line Tools**, which gives you `git` and `python3` in a single install. If you've ever used Terminal for development you probably already have it. To check:

```bash
git --version
```

If it prints a version, you're set. If it prompts you to install developer tools, click "Install" — that takes ~5 minutes once. (Or run `xcode-select --install` to trigger the prompt manually.)

Then in Terminal:

```bash
# 1. Clone the repo into your home folder
git clone https://github.com/Jonah-Keller/lecture-notes-viewer.git ~/lecture-notes-viewer

# 2. Put the launcher on your Desktop
ln -s "$HOME/lecture-notes-viewer/Launch Lecture Notes.command" ~/Desktop/
```

That's it. **Double-click `Launch Lecture Notes.command` on your Desktop.**

The first launch will:
1. Create a Python virtualenv and install dependencies (~1 minute)
2. Download Chromium for PDF export (~150 MB, one-time)
3. Prompt you in the Terminal window for your Anthropic API key
   (get one at https://console.anthropic.com/settings/keys)
4. Open the app in your browser at http://localhost:8080

Every subsequent click just runs the server (and silently pulls any updates from GitHub first).

To stop the server, press **Ctrl-C** in the Terminal window or close it.

## Day-to-day use

- Click the Desktop launcher to start.
- Click "New Course" in the app to create your first course.
- Upload one or more PDF + transcript pairs; notes generate in the background.
- Review drafts, publish, export PDFs.

## Troubleshooting

**"git: command not found"** — install Xcode Command Line Tools: `xcode-select --install`, click Install in the popup, wait, then re-try the clone.

**"Python 3.9+ is required"** — the launcher tells you this and exits. Install a newer Python from https://www.python.org/downloads/ and re-launch. Apple's bundled Python is usually fine.

**Launcher window closes immediately** — it shouldn't; it's set to stay open and show "Press Enter to close." If it still vanishes, run `~/lecture-notes-viewer/start.sh` directly from Terminal to see the error.

**Port 8080 already in use** — quit anything else using that port (or restart your Mac).

## Updating

The launcher silently runs `git pull` on each click, so improvements pushed to GitHub flow in automatically. To pull manually:

```bash
cd ~/lecture-notes-viewer && git pull
```
